# deai — 去 AI 味

> 理论来源：Strunk & White《The Elements of Style》、George Orwell《政治与英语》、中文写作风格经验

## 触发条件

以下任一情况：

- 用户说"读起来太 AI"、"像 AI 写的"、"去一下 AI 味"
- 用户说"帮我写得像人话"、"改得自然一点"、"不要那么官方"
- 在生成长文（>300 字）时，AI 可主动自查一遍再输出

## 5 类典型 AI 味（定位即可改）

### 类 1：套话开头

```
❌ 在当今时代，随着 XXX 的快速发展...
❌ XXX 是一个广泛讨论的话题...
❌ 毋庸置疑，XXX 具有重要意义...
```

**改法**：删掉，直接进入主题。好文章第一句就是主张或钩子。

### 类 2：结构词堆砌

```
❌ 首先... 其次... 再次... 最后...
❌ 综上所述，我们可以看到...
❌ 值得注意的是... 需要强调的是...
```

**改法**：
- 用具体段落替代"首先/其次"
- 删掉"值得注意"这类元语言，直接说那件值得注意的事
- "综上所述"后面的话通常是废话，整段删

### 类 3：空洞形容词

```
❌ 强大的 / 全面的 / 深入的 / 完善的 / 高效的
❌ 重要性 / 关键性 / 必要性（名词化）
```

**改法**：
- 用具体动词+具体数字替代："高效处理" → "30ms 处理完 10 万条"
- 形容词至少带一个量化：不是"很快"，是"快 3 倍"
- 名词化拆回动词句：不是"其重要性在于" → "重要在哪：..."

### 类 4：凑字并列

```
❌ 它不仅 A，而且 B，同时 C
❌ 既要 X，又要 Y，还要 Z
❌ 为 A、B、C 提供 X、Y、Z
```

**改法**：
- 并列超过 3 项就拆成列表
- 3 项之内的并列检查是否真并列（常有一项凑字）
- "不仅…而且"大多数情况可以删前半段，只留后半段

### 类 5：过度中性 / 骑墙

```
❌ 这是一个复杂的问题，需要综合考虑多方面因素
❌ 可能在某些情况下会有所不同
❌ 总体而言，利弊各半
```

**改法**：
- 有立场就亮立场："我选 A，原因是..."
- 把"可能"、"或许"、"在某些情况下"全删一遍，剩下的再看
- 真的需要中性时，给**具体的划分依据**："小项目用 A，大于 N 用户用 B"

## 改写流程

```
1. 通读一遍，标红 5 类 AI 味位置
2. 逐类改：
   - 套话开头 → 删到第一个具体句
   - 结构词 → 看是否真的需要结构
   - 形容词 → 加数字/例子/动词化
   - 并列 → 合并或列表化
   - 中性 → 亮立场或给切换条件
3. 读一遍——能想象"我在跟谁说话吗"？想不到就是还不像人
4. 大声读一遍——读起来卡的地方都是要改的地方
```

## 对照表（常见 AI 味 → 人话）

| AI 味 | 人话 |
|---|---|
| 综上所述 | （直接删） |
| 值得注意的是 | （直接说那件事） |
| 在当今时代 | （直接说具体时间或现状） |
| 具有重要意义 | 意义在于…（给具体内容） |
| 全面深入地分析 | 拆三块看：… |
| 一个复杂的问题 | 卡在…，主要原因是… |
| 多方面因素 | 3 个主要因素：A、B、C |
| 广受欢迎 | 上个月 1.2 万人用 |
| 取得了显著成效 | 从 X 降到 Y |
| 相关人员 | 谁（具体岗位/角色） |

## 反模式

❌ 改成方言、网络用语、油腻段子——不是去 AI 味，是换风格
❌ 把所有"的"全删——适度连词不是 AI 味
❌ 一味求短——该展开的地方还是要展开，AI 味不等于冗长
❌ 改完像翻译腔——注意保持中文语序

## 衡量标准

改完问自己 3 个问题：
1. 能想象说话人的具体身份/语气吗？
2. 随便挑一段，读起来有停顿、有呼吸、有语气吗？
3. 删掉每段第一句话，意思是否仍完整？（如果完整，说明那句就是 AI 套话）

## 与其他 skill 的组合

- 先用 `clarify` 把受众问清楚，再用 `deai` 按受众语气改
- 写完用 `challenge`（`?`）让 AI 自审一遍——AI 味在自己眼里最难发现

## 小作业

找一段 ChatGPT 的默认风格回答，自己动手按 5 类定位，改写一版。对比原文读。
